# autotrader 量化交易系统

## 项目概述

autotrader 是一个基于同花顺客户端的自动化交易系统，用于执行量化交易策略。该系统通过聚宽发送网络交易信号请求，并通过同花顺客户端模拟鼠标自动执行交易操作。基于jqktrader项目，添加了聚宽交易信号接收功能。

### 主要功能

- 接收和处理交易请求
- 自动执行买入、卖出操作
- 交易请求去重
- Token 验证
- 交易频率控制
- 错误处理和日志记录
- 持久化存储交易请求
- 重启后从上次未处理的请求开始执行

## 项目结构

```
jqktrader/
├── server.go          # Go语言编写的服务器，接收交易请求
├── trade_executor.py  # Python编写的交易执行器，执行交易操作
├── start_trading.ps1  # PowerShell启动脚本
├── README.md          # 项目说明文档
├── trade_requests.json    # 交易请求文件[自动生成]
├── processed_requests.txt # 已处理的请求ID文件[自动生成]
└── server_processed_requests.txt # 服务器已处理的请求ID文件[自动生成]
```

## 安装步骤

### 1. 安装依赖

#### Python 依赖
```bash
pip install jqktrader easyutils pandas pytesseract pypiwin32 pywinauto
```

#### Tesseract OCR
- 下载地址：https://github.com/UB-Mannheim/tesseract/wiki
- 安装后将 `tesseract.exe` 路径添加到系统环境变量

### 2. 编译服务器

```bash
go build -o server.exe server.go
```

### 3. 配置文件

- 确保同花顺客户端已安装并登录
- 确保 Tesseract OCR 已正确安装

## 使用方法

### 1. 启动系统

运行 PowerShell 脚本启动服务器和交易执行器：

```powershell
.\start_trading.ps1
```

### 2. 发送交易请求

向服务器发送交易请求，格式如下：

```json
[
  {
    "strategy": "MA1",
    "idx": 1,
    "time": "2026-03-16 10:00:00",
    "action": "buy",
    "zqdm": "600000",
    "qty": 100,
    "price": 10.0,
    "trade_time": "2026-03-16 10:00:00",
    "token": "xxxx"
  }
]
```

### 3. 交易执行

- 服务器接收到请求后，将其写入 `trade_requests.json` 文件
- 交易执行器监控该文件，执行未处理的交易请求
- 执行完成后，将请求标记为已处理

## 技术特点

### 1. 交易请求处理

- **去重机制**：相同的交易请求不会被重复处理，通过生成唯一请求标识实现
- **Token 验证**：只有 Token 正确的请求才会被处理，默认 Token 为 781818
- **频率控制**：确保 1 秒内只执行一次交易，避免操作过快导致的错误

### 2. 错误处理

- 完善的错误处理机制，确保系统稳定运行
- 详细的日志记录，便于排查问题
- 股票代码自动处理，确保是6位数格式
- 价格自动处理，保留三位小数

### 3. 可靠性

- 交易请求持久化存储，即使系统重启也能继续处理
- 已处理请求记录持久化，避免重复执行
- 单例模式管理交易客户端，避免重复初始化
- 线程安全设计，确保交易操作的原子性

## 注意事项

1. **同花顺客户端**：需要手动登录，系统不维护登录状态
2. **图形界面**：需要服务器有图形界面环境
3. **实盘风险**：该系统本质上是按键精灵，不能保证 100% 成功，请实盘时慎重使用
4. **性能优化**：对于大量交易请求，可能需要进一步优化系统性能
5. **路径问题**：确保 Tesseract OCR 和同花顺客户端路径正确
6. **依赖问题**：确保所有依赖包已正确安装

## 故障排除

### 常见问题

1. **服务器启动失败**：检查端口 8888 是否被占用
2. **交易执行失败**：检查同花顺客户端是否已登录，是否有足够的权限
3. **路径问题**：确保 Tesseract OCR 和同花顺客户端路径正确
4. **依赖问题**：确保所有依赖包已正确安装
5. **股票代码错误**：确保使用6位数的股票代码
6. **价格格式错误**：确保价格保留三位小数

### 日志查看

- 服务器日志：服务器窗口输出
- 交易执行器日志：交易执行器窗口输出
