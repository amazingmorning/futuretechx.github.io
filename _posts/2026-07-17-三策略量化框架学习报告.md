## 一、框架概述

本报告基于聚宽三策略的源码进行深度解析。该文件实现了一个**聚宽三策略合并框架**，采用子账户（SubPortfolio）模式运行三个独立的量化策略，共享同一账户资金但独立运作。

### 1.1 整体架构

| 子账户 | 策略名称 | 代码前缀 | 资金比例 | 基准 | 标的类型 |
|--------|---------|---------|---------|------|---------|
| pindex=0 | 七星172-拉普拉斯 | `A_` / `qixing_` | 50% | 沪深300 | ETF基金 |
| pindex=1 | 五福5.1V4 | `B_` / `wufu_` | 50% | 沪深300 | ETF基金 |
| pindex=2 | 小市值选股策略 | `C_` / `smallcap_` | 0%（禁用） | 深证综指 | 股票 |

> 当前配置中策略C资金比例为0，仅运行策略A和B。

### 1.2 核心设计模式

框架采用**状态隔离 + 统一调度**的设计模式：

- **`_StrategyState` 类**：每个策略拥有独立的状态对象（`g.A`、`g.B`、`g.C`），互不干扰
- **`_SubContext` 类**：通过代理模式隔离子账户的 portfolio 访问，使每个策略函数看到的 `context.portfolio` 只是自己那个子账户
- **`_run_with_state` 函数**：在调用各策略函数前切换全局 `g` 指向对应策略状态，调用后恢复

```python
# 状态切换核心逻辑（简化）
def _run_with_state(key, func, context):
    master = _master_state()
    state = getattr(master, key)  # 如 g.A
    old_g = g
    g = state  # 切换全局g为子策略状态
    sub_ctx = _SubContext(context, state.pindex)
    result = func(sub_ctx)
    g = old_g  # 恢复
    return result
```

## 二、策略A：七星172-拉普拉斯（ETF动量策略）

### 2.1 策略思路

策略A是一个**ETF动量轮动策略**，核心思想是：在正常市场状态下，持有动量得分最高的ETF；在震荡市场中，切换到防御性ETF（银华日利 511880.XSHG）。

### 2.2 ETF池

策略A维护一个约46只ETF的固定池，涵盖：
- 黄金ETF（518880）、QDII基金（513100/513500等）
- 行业ETF（券商512880、有色512400、通信515880等）
- 宽基ETF（沪深300/中证500/创业板等）
- 债券ETF（511880/511010/511220等）

### 2.3 评分体系

评分公式：**Score = 年化收益率 × R²**

```python
# 加权对数线性回归计算动量
y = np.log(recent_prices)
x = np.arange(len(y))
weights = np.linspace(1, 2, len(y))  # 近期权重更高
slope, intercept = np.polyfit(x, y, 1, w=weights)
annualized_returns = math.exp(slope * 250) - 1
r_squared = 1 - ss_res / ss_tot
score = annualized_returns * r_squared  # 动量×拟合度
```

**设计亮点**：
- 使用**对数价格回归**，避免极端值影响
- **线性递增权重**（1→2），近期数据权重更高
- **R²作为质量系数**，惩罚走势不稳定的标的

### 2.4 多重过滤器

策略A对候选ETF设置多重过滤：

| 过滤器 | 参数 | 作用 |
|--------|------|------|
| 盈利保护 | 回撤≥5% | 从近1日高点回撤5%即排除 |
| 溢价过滤 | >20% | 排除溢价过高的QDII基金 |
| 成交量过滤 | 比率>2倍均值 | 放量且年化>100%排除（追高风险） |
| 短期动量 | 10日年化<0排除 | 排除短期下跌标的 |
| 连跌过滤 | 任一日前跌幅>3% | 排除连续大跌标的 |
| 拉普拉斯滤波 | 正常期使用 | 价格需在拉普拉斯均线之上且斜率为正 |
| 高斯滤波 | 震荡期使用 | 价格需在高斯滤波值之上且斜率为正 |

### 2.5 市场状态判断（正常期 / 震荡期）

策略A通过**乖离率**和**RSI**判断市场状态：

**进入震荡期信号**：
- 乖离率 > 10%（价格偏离MA20过大）
- RSI超买回落（前日RSI>75，当日RSI<60）

**退出震荡期信号**（满足其一）：
- 从低点上涨 ≥ 3%
- 价格站上均线 + 回撤收窄 + RSI回升（稳定2天以上）
- 震荡期满15天（强制退出）

**状态切换冷却**：2个交易日内不重复切换

### 2.6 交易逻辑

```
卖出：不在目标ETF列表中的持仓 → 清仓
买入：目标ETF（排名第1），等权重分配
若无可选ETF → 切换到防御性ETF（511880）
```

## 三、策略B：五福5.1V4（ETF动量+相关性守卫）

### 3.1 策略思路

策略B是策略A的**增强版**，增加了更复杂的市场状态判断（三态）、相关性守卫机制、多重止损和回撤管理。

### 3.2 ETF池

策略B维护两个ETF池：
- **全球ETF池**（32只）：黄金、QDII、宽基等
- **中国ETF池**（98只）：行业、主题、宽基等
- 合并后约130只，通过**成交量动态过滤**

### 3.3 三态市场判断（正常期 / 震荡期 / 走弱期）

策略B使用**三指数均线系统**判断市场状态：

```python
# 三个基准指数
indexes = {'沪深300': '000300.XSHG', '深证综指': '399101.XSHE', '创业板': '399006.XSHE'}

# 判断逻辑
if below_ma20_count >= 4:    # 4个以上指数跌破MA20
    regime = '走弱期'
elif above_ma10_count >= 4:  # 4个以上指数站上MA10
    regime = '正常期'
else:
    regime = '震荡期'
```

**状态切换确认**：需连续2个交易日信号一致才切换，避免频繁抖动

**状态看板**：每日收盘后统计各状态的天数、累计收益、胜率
```
📈 【状态看板】正常期(0天 0.00% 胜0% 0+/0-/00) | 震荡期(22天 -11.74% 胜43% 9+/12-/00) | 走弱期(0天 0.00% 胜0% 0+/0-/00)
```

### 3.4 相关性守卫（核心创新）

策略B的**最大创新**是相关性守卫机制，防止在高相关ETF之间无效换仓：

**调整后相关系数**：
```python
# 不仅看收益率相关性，还考虑累计收益曲线差异和波动率比
base_corr = returns_df.corr()
curve_diff = np.mean(np.abs(cum_returns_A - cum_returns_B))
ret_factor = np.exp(-curve_diff)  # 曲线越像，因子越接近1
vol_ratio = min(vol_A, vol_B) / max(vol_A, vol_B)
adj_corr = base_corr * ret_factor * vol_ratio
```

**守卫规则**（按市场状态和动量分段）：
- 正常期：动量>7一律放行；中动量区间相关性>0.8-0.95拦截
- 震荡期：低动量区间相关性>0.3即拦截
- 走弱期：低动量区间相关性0.62-0.78拦截

**复盘机制**：记录每次拦截决策，次日对比持有vs候选的收益，验证拦截是否正确

### 3.5 多重过滤器

策略B的过滤体系比策略A更丰富，且**按市场状态差异化启用**：

| 过滤器 | 正常期 | 震荡期 | 走弱期 |
|--------|--------|--------|--------|
| 动量得分 | ✓ | ✓ | ✓ |
| R²拟合度 | ✓(0.39) | ✓(0.4) | ✗ |
| 均线MA10 | ✗ | ✗ | ✓ |
| 成交量 | ✓ | ✓ | ✓ |
| 短期风控(3%跌幅) | ✓ | ✓ | ✓ |
| 拉普拉斯滤波 | ✓(s=0.06) | ✓(s=0.05) | ✓(s=0.12) |
| 高斯滤波 | ✗ | ✓ | ✗ |
| 双正动量 | ✓(仅震荡期) | ✓ | ✗ |

### 3.6 止损体系

策略B实现**双重分钟级止损**：

1. **固定止损**：当前价 ≤ 成本价 × 91%
2. **百分比止损**：当前价 ≤ 昨收 × 95%

**止损后冷却**：2个交易日内不重新买入该标的

### 3.7 回撤管理

多级回撤阈值触发不同操作：

| 回撤幅度 | 操作 |
|---------|------|
| ≥3% | 进入监控 |
| ≥5%（走弱期）| 半仓 |
| ≥8%（走弱期）| 切换防御性ETF |
| ≥10% | 半仓 |
| ≥12%（走弱期）| 清仓 |
| ≥20% | 全部清仓 |

## 四、策略C：小市值选股策略

### 4.1 策略思路

经典的**小市值因子轮动策略**：每周一从深证综指成分股中筛选市值最小的10只股票持有。

### 4.2 选股逻辑

```python
# 1. 获取深证综指成分股
initial_list = get_index_stocks('399101.XSHE')

# 2. 过滤：排除ST、退市、创业板/科创板/北交所、涨停/跌停、上市不足375天
filtered = C_filter_stocks(context, initial_list)

# 3. 查询市值，升序排列，取前10只
query(valuation.market_cap.between(5, 300)).order_by(market_cap.asc())
final_list = df[:10]
```

### 4.3 特殊机制

- **空仓月**：1月和4月不持股，切换防御性资产
- **涨停打开卖出**：昨日涨停的股票，今日若打开涨停则卖出
- **双重止损**：个股跌幅≥10%止损 + 大盘平均跌幅≥5%止损
- **100%止盈**：个股收益达100%自动止盈

## 六、关键学习点

### 6.1 架构设计

1. **子账户隔离**：通过 `_SubContext` 代理模式实现策略间资金和持仓的完全隔离
2. **状态管理**：`_StrategyState` + `_run_with_state` 实现全局变量 `g` 的按策略切换
3. **统一调度**：所有 `run_daily` 在一个函数中集中管理，便于维护

### 6.2 量化技巧

1. **动量评分**：对数回归 + R²质量系数 + 线性递增权重
2. **市场状态**：多指数均线系统 + 状态切换确认机制
3. **相关性守卫**：调整后相关系数（曲线差异+波动率比）+ 分段规则
4. **多重止损**：固定止损 + 百分比止损 + 回撤管理 + 止损冷却

### 6.3 风控体系

1. **入场风控**：多重过滤器，按市场状态差异化启用
2. **持仓风控**：利润保护、分钟级止损、回撤管理
3. **系统风控**：防御性ETF切换、空仓机制、状态切换冷却

## 七、总结

该三策略框架展示了**专业级量化策略的工程实践**：

- **策略A**提供稳健的ETF动量轮动基准
- **策略B**通过相关性守卫和三态判断增强收益风险比
- **策略C**提供小市值因子暴露（当前禁用）

框架的**模块化设计**使得各策略可独立开发、测试和启停，通过资金比例配置即可灵活调整组合权重，是值得学习的多策略管理范式。
